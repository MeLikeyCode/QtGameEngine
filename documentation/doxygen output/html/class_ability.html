Detailed Description

An abstract class that represents an ability.

An Ability has an owner and an icon. The owner may be effected in some way when the ability is used, depending on the ability. The Icon is used for GUI purposes (i.e.by AbilityCell).

All Abilities will throw assertion errors if they are used after the owner is dead (this is automatically handled by NoTargetAbility, PointTargetAbility, and EntityTargetAbility). Most abilities require the owner to be in a Map before being used, but not all, so this is something that each Ability subclass should handle on its own. The main difference between Abilities and Items is the fact that Items are Entities, therefore they can be on the ground where as Abilities cannot.
